```
PRM - <ORIG> - <DISC> - <TYPE> - <NNNN>          [ + Rev + Status ]
 |      |        |        |        |
 |      |        |        |        4-digit sequential (per DISC+TYPE)
 |      |        |        Document type code (see §3)
 |      |        Discipline code (see §2)

```
Example: **`MGD-ATL-EL-SPC-0012`** Rev C / S4 → Atlas-originated Electrical Specification, seq 0012.

## 5. Revision Coding

Two-stage convention aligned to ISO 19650 CDE states:

| Stage | Rev Series | Meaning |
|---|---|---|
| Pre-issue / design development | **P01, P02 …** | Work-in-progress, author environment only (Status S0) |
| Issued for review / approval | **A, B, C …** | Shared for coordination / review / approval (Status S1–S4) |
| Issued for Construction / Use | **0** | Authorised, published, contractual (Status A1) |
| Post-issue revisions | **1, 2, 3 …** | Re-issued after Rev 0 (A1, or B1 if issued with comments) |

## 6. ISO 19650 Suitability (Status) Codes Used
